## 二、SOP 主流水线 (核心选品自动化)

由 **SOP-主调度器** (`hi3J0nArXh0BEoqO`) 每天北京时间 10:00 自动触发，按顺序执行 8 个子步骤:

```
SOP-主调度器 (cron: 0 2 * * * = 北京10:00)
│
├── 0. 刷新eHunt-Token ──→ eHunt-Token自动刷新
│
├── 1. SOP-步骤1-自动挑类目 (iOL9f9tbQJNKkFBP)
│      作用: 从eHunt获取品类/产品/畅销品排行 → AI分析推荐6~10个蓝海类目
│      输出: 推荐类目列表 → 存入 categories 表
│
├── 2. SOP-步骤2-关键词挖掘 (ZWmD4UFMsnz3vQdr)
│      作用: 对每个推荐类目 → eHunt获取关键词 → AI生成/评分/蓝海筛选
│      输出: S/A/B/C级关键词 → 存入 keywords 表
│
├── 3. SOP-步骤3-蓝海评分 (UM24AFvqv9CEnIK8)
│      作用: 对关键词从4个eHunt数据维度获取数据 → 计算蓝海分数
│      输出: 蓝海评分结果 → 更新 keywords 表
│
├── 4. SOP-步骤4-反查产品 (UzmEtfTxsLI36NWc)
│      作用: 对A+B级关键词 → eHunt搜索产品 → AI分类过滤
│      输出: 筛选后的产品列表 → 存入 products 表
│
├── 5. SOP-步骤5to9-深度分析 (36mx5obPYV67EsJP)
│      作用: 对产品进行5个维度深度分析
│        Step5: Listing对比分析 (竞品分析)
│        Step6: 店铺竞争格局分析
│        Step7: 广告投放分析
│        Step8: 失败模式分析 (下架/差评原因)
│        Step9: 跨平台分析 (Amazon手工品对比)
│      输出: 多维度分析报告 → 存入 product_analysis 表
│
├── 6. SOP-步骤10to12-AI生成 (qqE9oSwkjPOQA0mO)
│      作用: AI生成选品建议/产品描述优化/定价策略
│      输出: AI生成的选品报告
│
└── 7. SOP-步骤13-日报生成 (Nbq2vaLRWheyrUdc)
       作用: 汇总当天SOP执行结果 → 生成日报
       输出: 日报邮件
```

## 四、基础设施工作流

| 工作流 | 触发方式 | 作用 |
|--------|----------|------|
| **eHunt-Token自动刷新** (`D0o0Q952jalrewzd`) | 被主调度器调用 | 检查JWT是否过期→模拟浏览器登录→获取新Token→保存到DB |
| **通用-配额检查与扣减** (`5HJvfUiZcGRwNWsO`) | 被业务流调用 | 查询当日配额→判断是否允许→扣减配额→返回结果 |
| **邮件通知** (`PV4iVgFLPpd7IdwS`) | 被业务流调用 | 按模板路由(日报/周报/月报/告警/错误)→渲染HTML→发送邮件→记录日志 |
| **全局错误通知** (`utsy0MDaVwJYwaAj`) | 任何工作流出错 | 格式化错误信息→调用邮件通知发送告警邮件 |
| **配额每日重置** (`kirP8eODCFZoBfQb`) | 每天00:00(北京) | 从quota_config复制今日配额→重置used_today=0→记录日志 |

## 五、完整调用关系图

```
SOP-主调度器 ─────────┬──→ eHunt-Token自动刷新
                       ├──→ SOP-步骤1-自动挑类目
                       ├──→ SOP-步骤2-关键词挖掘
                       ├──→ SOP-步骤3-蓝海评分
                       ├──→ SOP-步骤4-反查产品

                       ├──→ SOP-步骤10to12-AI生成
                       ├──→ SOP-步骤13-日报生成

竞品动态监控 ───────────→ 邮件通知
周报月报生成 ───────────→ 邮件通知
全局错误通知 ───────────→ 邮件通知

所有业务流 ────────────→ 通用-配额检查与扣减 (配额控制)
```

```
                │                      │                      │
                └── sop_executions     └── quota_usage        └── product_analysis
                    (执行日志)            (配额记录)              (分析报告)
```
